Setting up a Personal Site Using Hugo and Cloudflare Pages

To get a site up quickly I am going to Hugo

Installing Hugo for Local Development#

NOTE The instructions provided in this document will be for Ubuntu 24.04. Installation instructions for other operating systems are available here.

Following best practices of not testing in production, I will be setting up a local development environment for my Hugo site. Step-by-step instructions are provided below.

Installing Hugo#

Luckily there is already an apt package for Hugo so no need to add additonal repositories or anything like that. Just simply install using apt. As of this writing the version installed via apt is 0.151.

1

$ sudo apt install hugo

Create your site directory#

Hugo’s cli allows for the easy creation of new sites using the new site command. I will be naming my site directory fjoenichols-dot-com because the domain for my website is fjoenichols.com.

1

$ hugo new site fjoenichols-dot-com

NOTE If you prefer to use yaml for configuration instead of toml you can append the new site command with --format yaml hugo new site <your-site-name> --format yaml

Once the site directory is created it should look something like the following

1
2

$ ls
archetypes  assets  content  data  hugo.toml  i18n  layouts  static  themes

Add a Theme#

The first thing you will want to do after creating the site directory is to identify a suitable theme for your Hugo site. There are a number of prebuilt themes available from the Hugo community at themes.gohugo.io.

I will be using the PaperMod theme as it is fairly minimal and I like the style.

The recommended way of installing PaperMod is as a git submodule. Since I’m planning to keep the site’s code on GitHub I will go ahead and initialize the site directory as a git repository.

To use the theme with the site I need to edit the hugo.toml file to add the theme. While in the hugo.toml file I will go ahead and set the baseURL and title.

NOTE Some theme installation instructions might say to edit config.toml. As of version 0.110, hugo.toml is the default config file.

1
2
3
4

baseURL = 'https://fjoenichols.com/'
languageCode = 'en-us'
title = 'F. Joe Nichols'
theme = ["PaperMod"]

Adding Taxonomies for 21-Day Sprints#

To support my 21-Day Sprint methodology, I need to tell Hugo and PaperMod how to group my posts. I’ll add the following to hugo.toml:

1
2
3
4

[taxonomies]
    category = "categories"
    tag = "tags"
    sprint = "sprints"

Deploying to Cloudflare Workers#

Now that the local environment is verified using hugo server -D, it’s time to push to production.

Create a Wranger TOML file#

Cloudflare Workers use wrangler as part of their build process. Luckily the build images already have Hugo built in so we just need to add a file called wrangler.toml to our project’s root directory.

1
2
3
4
5

name = "fjoenichols-dot-com"
compatibility_date = "2026-01-10"

[assets]
directory = "./public"

Push to GitHub#

Cloudflare Pages works best with a git-integrated workflow.

1
2
3
4

$ git add .
$ git commit -m "Initial site setup with PaperMod"
$ git remote add origin https://github.com/fjoenichols/your-repo-name.git
$ git push -u origin main

Configure Cloudflare Worker Deployment#

This is a lot of point and click working through the Cloudflare UI.

I was looking for Workers & Pages and I couldn’t find it so I search for Pages instead and then it popped up.

Then I clicked on Create application and Continue with GitHub

On the Create a Worker screen I selected my GitHub repository and left the Project name the same.

For Build command I used hugo --gc --minify and left npx wranger deploy in the Deploy command.

Then I clicked “Deploy”

At this point Cloudflare provisioned a container with my static files generated by hugo and give me a workers.dev URL.

Mapping your Domain#

I register my domains through Cloudflare so this is pretty straight forward.

From the newly deployed application dashboard click “Settings” and click + Add next to “Domains & Routes”

Domains & Routes will pop up on the right side of the screen and select “Custom Domain”. Here I enter my domain name and click Add Domain at the bottom of the screen.
